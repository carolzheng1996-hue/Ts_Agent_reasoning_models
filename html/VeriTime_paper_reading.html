1. 一句话总结

VeriTime 不只监督最终答案，还监督模型是否正确识别任务意图、关键模式、初步答案和自检结果；它用 TSRgen 合成 process-verifiable CoT 数据，并用难度调度 + 多目标 GRPO，让 3B-4B 级 LLM 在多类时序推理任务上显著提升。

2. TSRgen 数据合成

TSRgen 用规则抽取器筛选时间序列任务，用 DeepSeek-R1 生成结构化 TS-tailored CoT，并从最终答案正确的轨迹中抽取过程标签。

Step名称作用
1Analyzing task intent识别题目目标、实体、要求
2Selecting key patterns提取趋势、阈值、幅度、波动、领域关键词等关键模式
3Analyzing time series samples用关键模式分析整段或关键片段
4Generating preliminary answer给出初步结论
5Reflection / backtracking检查模式是否完整、排除干扰因素
6Final summary汇总推理并输出最终答案

4. 数据调度与训练

训练数据定义为：

$$\mathcal{D}_{train}=\{(x_i,y_i,\mathcal{L}_i)\mid i=1,2,\dots,n\}$$

$x_i$ 是问题，$y_i$ 是 CoT，$\mathcal{L}_i=\{l_{\text{final}},l_{\text{process}}\}$ 包含最终答案和过程标签。

VeriTime 先用 10% 数据 warmup SFT，然后用 warmup model 把 scenario 数据分为 normal 和 hard。normal 用 SFT，hard 用 RL：

$$\theta_{final}=\arg\max_{\theta}\mathbb{E}_{(x_i,\mathcal{L}_i)\sim\mathcal{D}_{hard}}\left[\mathcal{R}(\pi_{\theta}(x_i),\mathcal{L}_i)\right]$$

8. 迁移、效率与消融

TimeSeriesExam 上，Qwen3/Qwen2.5 分别提升 24.46% 和 16.85%；DROP 上，Qwen3/Qwen2.5 分别提升 8.36% 和 52.87%。VeriTime 还显著减少推理 token，Qwen3-4B 在 ECG 任务从 7,093 tokens 降到 965，减少 86.40%。

消融显示 hard reward 和 process reward 都关键；去掉 hard reward 时 inferential calculation 从 68.6% 降到 47.6%。去掉 critical pattern reward 平均下降 18.94%，说明识别关键时序模式是最终正确的核心。

10. 贡献与局限

VeriTime 的贡献是把时间序列 reasoning RL 从 final-answer reward 推到 process-verifiable reward，并证明数据调度能提高效率。局限是代码仍是匿名链接，主要实验集中在 compact LLM；过程标签依赖 DeepSeek-R1 与规则抽取，后续需要更广领域和更强 verifier。
