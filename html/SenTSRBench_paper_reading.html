1. 一句话总结

这篇论文提出一种“把时间序列专家模型的领域分析注入到通用推理模型思维链里”的框架，并发布 SenTSR-Bench：一个来自真实工业传感器场景、含多阶段诊断问题的多变量时间序列推理 benchmark。

2. 问题背景

真实工业维护中的时间序列诊断需要回答 What happened、How happened 和 Suggested fix 三类问题。论文指出，通用推理模型 GRLM 推理强但缺少传感器领域知识，时间序列专家模型 TSLM 更懂局部波形但复杂推理弱。因此，论文把 TSLM 的领域观察注入 GRLM 的 reasoning trace，让二者分工。

模型类型优点缺点
GRLM复杂推理、上下文整合能力强容易误读时间序列模式
TSLM更擅长识别趋势、异常、传感器模式泛化推理和诊断链较弱

3. 任务定义

文本问题和多变量时间序列分别写作：

$$\mathbf{q}=(q_1,\ldots,q_n)\in V^*,\qquad \mathbf{X}=\{\mathbf{x}_t\}_{t=1}^{T},\quad \mathbf{x}_t\in\mathbb{R}^{D}$$

推理模型先生成内部轨迹 $\mathbf{r}$，再生成答案 $\mathbf{y}$：

$$\pi(\mathbf{r}\mid \mathbf{X},\mathbf{q}) =\prod_{k=1}^{K}\pi(r_k\mid \mathbf{X},\mathbf{q},[\langle\mathrm{think}\rangle,\mathbf{r}_{这个形式把 reasoning trace 暴露成可注入对象，而不是把模型当成只能接受外部 prompt 的黑盒。

4. Knowledge Injection

框架包含冻结通用推理模型 $\pi^G$ 和时间序列专家模型 $\pi^T$。先基于问题和当前 GRLM 前缀构造专家查询：

$$\tilde{\mathbf{q}}=\mathsf{Query}(\mathbf{q},\mathbf{r}^{G}_{\le k})$$

TSLM 生成领域知识：

$$\mathbf{K}^{T}\sim\pi^{T}(\cdot\mid \mathbf{X},\tilde{\mathbf{q}})$$

再把这段知识插入 GRLM 推理轨迹：

$$\mathbf{r}^{\mathrm{Inj}}_{\le k} =\mathsf{Inject}(\mathbf{r}^{G}_{\le k},\mathbf{K}^{T})$$

GRLM 从注入后的轨迹继续生成：

$$r^{G}_{j}\sim \pi^{G}(\cdot\mid \mathbf{X},\mathbf{q},[\langle\mathrm{think}\rangle,\mathbf{r}^{\mathrm{Inj}}_{5. 注入策略
策略插入位置做法结论
Early<think> 后TSLM 先给时间序列观察，GRLM 继续反思推理默认方案，整体最好
Intermediate低置信步骤处TSLM 校验或纠正某个低置信句子有效但更复杂
Late完整草稿后TSLM 审阅完整 reasoning trace有效但更容易偏离专家模型训练分布

Early injection 写作：

$$\mathbf{r}^{\mathrm{Inj}}_{\le 1} =\mathsf{Inject}_{reflect}(\emptyset,\mathbf{r}^{T}) =[\mathbf{r}^{T},\mathbf{v}_{reflect}]$$

6. RL-Honed Thinking

普通 TSLM 往往只会直接答题，而 knowledge injection 需要的是可注入的中间分析。论文用 RLVR 让 TSLM 学会先输出 thinking trace，再输出 answer。completion 写作：

$$\mathbf{z}=[\mathbf{r},\mathbf{y}]$$

GRPO 目标为：

$$\max_{\theta}\; \mathbb{E}_{\{\mathbf{z}_i\}_{i=1}^{G}\sim\pi_{\theta}(\cdot\mid\mathbf{X},\mathbf{q})} \left[ \mathcal{L}_{GRPO}\left(\theta,\{R(\mathbf{z}_i)\}_{i=1}^{G}\right) \right]$$

奖励由格式和最终答案组成：

$$R(\mathbf{z})=r_{\mathrm{fmt}}(\mathbf{z})+r_{\mathrm{hard}}(\mathbf{z})$$

组内 advantage 标准化为：

$$\hat{A}_i=\frac{r_i-\mu_r}{\sigma_r},\quad \mu_r=\frac{1}{G}\sum_{j=1}^{G}r_j,\quad \sigma_r=\sqrt{\frac{1}{G}\sum_{j=1}^{G}(r_j-\mu_r)^2+\gamma}$$

这种训练不需要人工标注中间推理链，只需要输出格式和答案正确性可验证。

7. SenTSR-Bench 数据集

SenTSR-Bench 是 Sensor-based Time-Series Diagnostic Reasoning Benchmark，来自真实工业机器监控环境，包含 vibration acceleration、velocity、temperature 等多变量传感器信号。

项目数量 / 描述
初始候选样本约 2,000 条多变量传感器序列
最终评测序列110 条真实、去标识化 multivariate sensor streams
评测问题330 道 MCQ
问题阶段What happened / How happened / Suggested fix
标注human-curated diagnostic annotations

训练数据由 23 条去标识化 seed 序列出发，先让 VLM 写 Python simulator 模拟相似行为，再随机化参数生成 6,000 条 MCQ 训练样本。

13. 消融与分析

完整注入需要保留原始时间序列。 只给 GRLM TSLM 文本摘要能带来约 7% 平均提升，但让 GRLM 同时访问原始时间序列和注入摘要可带来约 17% 平均提升。

论文还比较 prompting 与 injection：把 TSLM 分析作为外部 prompt 不如把它放入 reasoning trace。原因是后者更容易让 GRLM 在推理过程中持续使用和反思领域知识。

训练数据多样性实验显示，没有 synthetic data 时小 TSLM 接近随机；使用约 50% seed-generated 数据即可恢复大部分最终性能，超过 75% 后收益变小。RL objective 对比中，DAPO 等方法收敛更平滑，但最终 reward 接近 GRPO。

14. 贡献与局限

主要贡献包括：提出面向时间序列诊断的 knowledge injection 框架；用 RLVR 训练无需人工思维链标注的 analysis-first TSLM；发布真实工业传感器诊断 benchmark SenTSR-Bench；在 SenTSR-Bench、TSEvol、TS&Language 上验证 injection 同时优于独立 TSLM 和独立 GRLM。

主要局限是：SenTSR-Bench 公开下载入口当前未确认；训练集依赖 VLM simulator，真实长尾故障覆盖仍需验证；闭源模型接口是否支持 assistant prefill 会影响落地；Suggested Fix 的收益不如 What/How 稳定，说明维修建议还需要更多外部知识和领域约束。
